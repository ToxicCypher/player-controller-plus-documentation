Character Controller Plus : State Definitions

Created by Cameron Gibson, last modified on Apr 19, 2024

Summary

In this section, a table of the available player states is provided to explain in moderate detail the purpose of each available state.

General Entry Conditions only explain the bare circumstances that a player can enter a given state. All states can be configured to fit the needs of the developer.

For more details regarding each states entry conditions, please view the State Definitions > [Target State] documentation.

Available States

Given the level of control that the end-user has over the configuration for this asset, separate documentation has been written that targets specific states to further explain the general usage and configurations available for each target state.

Globally shared configuration parameters can be found in the State Manager Base State documentation although, any globally shared configuration that pertains directly to any specific state, globally shared configuration will be documented in the concerned state documentation.

State

State Summary

High Level Entry Conditions

Idle

The player is grounded while being unaffected by directional movement.

  • Grounded

  • Absent Direction Input

Walking

The player is grounded while being affected by directional movement that falls under a set threshold. Movement in this state is slower than when in a Running state.

  • Grounded

  • Directional Input w/ a Magnitude < 0.5

Running

The player is grounded while being affected by directional movement that falls above a set threshold. Movement in this state is faster than when in a Walking state.

  • Grounded

  • Directional Input w/ a Magnitude >= 0.5

Skidding

The player is grounded and has changed direction abruptly causing the character to skid to a stop before continuing movement in the re-targeted direction.

  • Grounded

  • Meets a Speed Threshold

  • Must be in a Running state

  • Direction Input Abruptly Polarizes

Jumping

The player is not grounded and travels upwards until the player releases jump input or the apex of the jump has been reached.

  • Grounded

  • Jump Input Pressed (Held)

Secondary Jump

The player is not grounded and travels upwards until the player releases jump input or the apex of the secondary jump has been reached. This state is used for many cases such as double jumping, jumping from a climb, jumping out of water, etc.

  • Not Grounded

  • Jump Input Pressed (Held)

Airborne

The player is traveling downwards due to the effects of gravity. This state can be entered when exiting a jump of when simply walking/running/rolling/etc. from a ledge.

  • Not Grounded

Airborne Advance

The player advances forward, unaffected by gravities downward force. This state can simply be characterized as an airborne dash but has been provided with configuration to extend the possibilities of what this state can be used for.

  • Not Grounded

  • Airborne Advance Input Pressed

Falling

This state is entered when the player has been in an Airborne state for a configured length of time.

  • Not Grounded (For a configured length of time)

Falling To Splat

This state is entered after a configured length of time that the player has been falling. When this state is entered, upon becoming grounded, the player will enter a Splatting state.

  • Not Grounded (For a configured length of time)

Soft Landing

This state is entered after a configured length of time that the player has been falling. When this state is entered, upon becoming grounded, the player will enter a Soft Landing state.

  • Grounded

  • Must Fall for Determined Time

Hard Landing

This state is entered after a configured length of time that the player has been falling. When this state is entered, upon becoming grounded, the player will enter a Hard Landing state.

  • Grounded

  • Must Fall for Determined Time

Splatting

This state is entered after a configured length of time that the player has been falling. The player must be in a Falling To Splat state to enter this state upon being grounded.

  • Grounded

  • Must Fall for Determined Time

Getting Up

This state is entered only after the player has splatted or after the player has slid from a slope in a slope facing position.

  • Grounded

  • Must be from Splatting or Sliding States.

Grounded Advance

The player moves (roll/dash) forward depending on the configuration as well as the provided animation.

  • Grounded

  • In Running State

  • Speed Threshold Met

Crouching

This state slows the movement of the player and locks into the crouching state until state changing conditions are met or the player exits the crouching state by entering crouch input again.

  • Grounded

  • Crouch Input Pressed

Sliding

This state sends the player sliding down any slope that the user has configured as such, whether it be contoured, flat or segmented.

  • Grounded

  • On Slope of Configured Angle

Climbing

This state allows the player to vertically and horizontally climb any surface that meets the Layer requirements that are set by the end-user. This state allows players to climb around surfaces that are cylindrical, such as poles.

  • Player Contacts “Climbable” Surface

Ledge Jumping

This state is entered when the apex of a climb has been reached, thrusting the player into a Secondary Jumping like state to hoist the player over the face of the climb.

  • Climbing

  • Peek of Climb is Reached

Ledge Climbing

This state is entered when the apex of a climb has been reached, performing a ledge climbing-like motion to place the player on the climbable surface.

  • Climbing

  • Peek of Climb is Reached

Swimming

This state is entered when the player comes into contact with a determined body of liquid. The player swims along the surface of the liquid.

  • Player Contacts “Swimmable” Surface

Diving

The player dives downward until the dive has completed its course or the player hits a surface, sending the player back upwards to the surface.

  • Swimming
